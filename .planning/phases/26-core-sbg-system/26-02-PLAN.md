  <action>
Add function to extract perimeter vertices from a set of grid cells:

```typescript
function extractPerimeter(cells: Set<string>, cellSize: number): Point[]
```

Algorithm:
1. For each cell, check 4 edges (top, right, bottom, left)
2. An edge is "perimeter" if the adjacent cell is NOT in the set
3. Collect perimeter edges as line segments
4. Order segments into a continuous polygon (walking the perimeter)
5. Subdivide long edges to add vertices (target ~15-20px spacing for smooth curves)
6. Return points in clockwise order

Edge case handling:
- Handle holes in groups (cells surrounded by group but not in it) - ignore inner perimeters for now
- Handle single-cell groups (4 perimeter edges, square shape)
- Handle L-shapes, T-shapes, complex polyominoes

Add to demo: button to test with a hardcoded 4-cell T-shape group.

AVOID: Don't try to smooth corners yet (that's rendering). Don't handle multiple disconnected groups in one call.
WHY: Keep extraction clean; smoothing is render concern; groups are always connected by definition.
  </action>
  <verify>Add test button that creates a T-shape from 4 cells, verify perimeter has correct outline (should trace around the T)</verify>
  <done>extractPerimeter returns ordered perimeter vertices for arbitrary cell groups</done>
</task>

<task type="auto">
  <name>Task 2: Create body generation from perimeter</name>
  <files>src/components/SoftBodyDemo.tsx</files>
  <action>
Add function to create a complete Body from perimeter vertices:

```typescript
function createBodyFromPerimeter(perimeter: Point[], color: string): Body
```

Steps:
1. Calculate centroid of perimeter (average of all points) - this is the hub
2. Add hub as last point in points array
3. Create springs:
   - Adjacent: connect consecutive perimeter points (+ wrap from last to first)
   - Spokes: connect hub to each perimeter point
   - Skip-2: connect every other perimeter point (structural cross-bracing)
   - Skip-4: if >8 perimeter points, add skip-4 springs
4. Calculate rest volume using shoelace formula
5. Initialize all velocities to 0

Use same spring constants as existing hardcoded bodies.

Update demo:
- Add button "Generate T-Body" that uses extractPerimeter + createBodyFromPerimeter
- Replace one of the hardcoded bodies with a generated T-shape body
- Verify it has same physics behavior (falls, bounces, collides)

AVOID: Don't generate springs for very close points (<5px) - causes instability.
WHY: Very short springs have extreme forces that blow up physics.
  </action>
  <verify>Generated T-body falls, bounces, maintains shape, collides with other body same as hardcoded bodies</verify>
