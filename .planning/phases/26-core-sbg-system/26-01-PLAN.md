**From Phase 25 Research - Hard Problem #6:**
Body-to-body collision requires:
- Detecting perimeter-to-perimeter penetration
- Pushing penetrating vertices out along collision normal
- Both bodies should deform (not just one)
- Performance concern: O(n*m) vertex checks per frame

**Current state:**
- Demo has 2 bodies but they don't interact
- Blue body has ground at y=400 (doesn't collide with red)
- Ground collision working with bounce/friction

**Physics constants from prototype:**
```typescript
BOUNCE = 0.3
FRICTION = 0.85
```

  <name>Task 1: Implement edge-based collision detection</name>
  <files>src/components/SoftBodyDemo.tsx</files>
  <action>
Add a collision detection system between two soft bodies:

1. For each perimeter edge on body A, check if any perimeter vertex of body B is inside body A
2. Use point-in-polygon test (ray casting or winding number)
3. For penetrating vertices, find the closest edge and compute the penetration depth and normal
4. Apply separation force to push vertex out (and equal-opposite force to edge endpoints)

Implementation approach:
- Add `checkBodyCollision(bodyA: Body, bodyB: Body)` function
- Use edge normals pointing outward (computed from perimeter winding order)
- Penetration response: move vertex along normal by half penetration depth, apply impulse

AVOID: Don't check hub points (only perimeter). Don't use broad-phase yet (only 2 bodies).
WHY: Hub collision creates unrealistic behavior; broad-phase adds complexity for no gain with 2 bodies.
  </action>

<task type="auto">
  <name>Task 2: Tune collision response and stability</name>
  <files>src/components/SoftBodyDemo.tsx</files>
  <action>
Adjust collision response to feel natural:

1. Add collision softness parameter (0-1) to control how much bodies "sink" before pushing back
2. Apply velocity-based friction at contact points (bodies sliding against each other)
3. Add position correction to prevent slow drift through (Baumgarte stabilization or direct position fix)
4. Ensure symmetric response: both bodies receive equal-opposite forces

Tune values:
- Collision softness: ~0.1-0.2 (allow slight overlap for soft feel)
- Collision friction: similar to ground FRICTION (0.85)
- Position correction: 0.2-0.5 of penetration per frame

AVOID: Don't make collision perfectly rigid (looks wrong for jello). Don't over-correct (jitter).
WHY: Soft bodies should squish slightly before bouncing; over-correction causes oscillation.
  </action>
